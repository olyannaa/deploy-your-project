## 1. Общие сведения

Модуль «Финансы» предназначен для полного учёта и контроля финансовых потоков проектной организации. Он объединяет:

- **Выплаты** — зарплаты, авансы, оплата субподрядчикам, дополнительные затраты.
- **Приходы** — поступления средств от заказчиков.
- **Бюджетный контроль** — отслеживание остатков бюджета с цветовой индикацией.
- **Расчёт ЗП** — автоматическое распределение зарплатных расходов по проектам.
- **Бухгалтерские задачи** — задачи на выплаты, привязанные к проектам или без привязки.

## 2. Роли и права доступа

В системе три ключевые роли для финансового модуля:

### Таблица доступа

| Возможность | Администратор | ГИП (Главный инженер проекта) | Бухгалтер |
|---|:---:|:---:|:---:|
| Страница «Финансы» в боковом меню | ✅ (только чтение) | ❌ (не видит в меню) | ✅ (чтение + редактирование) |
| Вкладка «Финансы» в карточке проекта | ✅ (только чтение) | ✅ (только чтение) | ✅ (чтение + редактирование) |
| Внесение выплат (клик по ячейке таблицы) | ❌ | ❌ | ✅ |
| Внесение приходов (клик по ячейке таблицы) | ❌ | ❌ | ✅ |
| Создание бухгалтерских задач | ✅ | ✅ | ✅ |
| Проведение ведомостей ЗП | ❌ | ❌ | ✅ |
| Просмотр всех проектов в таблице | ✅ | ❌ (только свои) | ✅ |
| Домашняя страница по умолчанию | Проекты | Проекты | Финансы |

### Важные ограничения по ролям:

**Бухгалтер:**
- Страница «Финансы» — домашняя.
- Видит ВСЕ проекты (не ограничен подразделением).
- Может вносить выплаты и приходы.
- На странице «Задачи» видит только задачи, где является исполнителем.
- В карточке задачи НЕ видит поле «Раздел WBS».
- На странице «Учёт рабочего времени» доступны только собственные задачи, вкладка «Экономический анализ» скрыта.

**ГИП:**
- НЕ видит пункт «Финансы» в боковом меню.
- Видит вкладку «Финансы» внутри карточки каждого своего проекта (только чтение).

**Администратор:**
- Видит всё в режиме чтения.
- Может создавать бухгалтерские задачи.

## 3. Модуль «Финансы» (глобальная страница)

**Расположение:** Боковое меню → «Финансы»  
**URL:** `/finance`  
**Доступ:** Администратор, Бухгалтер

Страница содержит **три вкладки** (tabs):
1. **Финансы** — основная сводная таблица
2. **Распределение ЗП** — начисления по сотрудникам
3. **Выплаты ЗП** — ведомости выплат

---

### 3.1. Основная таблица (вкладка «Финансы»)

#### 3.1.1. Общая структура

Таблица представляет собой матрицу:
- **Строки** = проекты + раздел «Вне проекта»
- **Столбцы** = аналитические колонки + колонки дат (недели) + колонки ЗП (фиксированные даты)

#### 3.1.2. Переключатель «Факт / План»

В верхней части таблицы — переключатель режима:
- **Факт** — показывает фактические выплаты и приходы (режим по умолчанию).
- **План** — показывает планируемые приходы (подробнее в разделе 8).

#### 3.1.3. Настройка видимости столбцов

Кнопка с иконкой «шестерёнка/слайдеры» открывает popover со списком аналитических столбцов. Каждый столбец можно включить/выключить чекбоксом.

#### 3.1.4. Аналитические столбцы (фиксированные, слева)

| № | Название столбца | Подсказка (тултип при наведении) | Формула расчёта |
|---|---|---|---|
| 1 | **Сумма по договору** | «Бюджет проекта» | Берётся из поля `budget` карточки проекта |
| 2 | **Поступления** | «Поступления по заключённым договорам в рамках проекта» | Сумма всех записей приходов (`incomes`) по этому проекту за всё время |
| 3 | **Сальдо (Остаток к поступлению)** | «Разница между Суммой по договору и Поступлениями» | `Сумма по договору − Поступления` |
| 4 | **Операционные расходы** | «Заключённые с Субподрядчиками/Фрилансерами + Доп. затраты» | Сумма выплат с reason = `subcontract` + `additional` по этому проекту |
| 5 | **ФОТ** | «Разница между Расходом и Оплачено» | Значение из функции `getProjectFOT(projectId)` — сумма `дни × дневная_ставка` всех сотрудников, работающих на проекте |
| 6 | **Оплачено (с ЗП)** | «Сумма всех выплат по проекту» | Операционные расходы + распределённые ЗП-выплаты по проекту |
| 7 | **Сальдо (Между поступлениями и расходами)** | «Разница между столбцом Поступления и Оплачено (с ЗП)» | `Поступления − Оплачено (с ЗП)` |

**Цветовая индикация столбца «Сальдо (Остаток к поступлению)»:**

| Цвет | Условие | Значение |
|---|---|---|
| 🟢 Зелёный | Остаток > 50% от бюджета | Норма |
| 🟡 Жёлтый | Остаток 10–50% от бюджета | Внимание |
| 🔴 Красный | Остаток < 10% от бюджета | Критично |

**Тултипы на заголовках:** При наведении на название любого аналитического столбца показывается всплывающая подсказка с кратким описанием (поле «tooltip»). Заголовок визуально подчёркнут пунктиром (`border-dotted`), курсор `cursor-help`.

#### 3.1.5. Колонки дат (недели)

Столбцы группируются по месяцам. Период отображения — **текущее полугодие** (январь–июнь или июль–декабрь).

Каждая недельная колонка определяется так:
- Дата = начало недели (понедельник).
- Заголовок = `dd.MM` (например, `06.01`, `13.01`).
- Месячная группировка в верхней строке заголовка (`Январь 2026`, `Февраль 2026`...).

**Структура ячейки (для каждого проекта в каждой неделе):**
```
┌──────────────┐
│  −120 000 ₽  │  ← Расходы (красный текст, знак «−»)
│  +500 000 ₽  │  ← Приходы (зелёный текст, знак «+»)
└──────────────┘
```

- Верхняя строка — сумма РАСХОДОВ за эту неделю (все `payments` по проекту с индексом этой недели).
- Нижняя строка — сумма ПРИХОДОВ за эту неделю (все `incomes` по проекту с индексом этой недели).
- Пустые значения не отображаются (ячейка пустая).

**При наведении на ячейку (тултип):** показать детализацию — список транзакций:
- Категория (ЗП/Аванс, Субподрядчики, Доп. затраты, Другое)
- Название задачи (если привязана)
- Сумма

**При клике на ячейку (только для бухгалтера):** открывается диалог добавления выплаты/прихода (подробнее ниже).

#### 3.1.6. Колонки ЗП (фиксированные даты)

Между недельными столбцами вставляются **фиксированные даты зарплатных выплат**:
- **25-е число** каждого месяца — **Аванс** за текущий месяц
- **10-е число** каждого месяца — **ЗП** (остаток) за предыдущий месяц

Эти колонки вычисляются автоматически и вставляются в хронологическом порядке среди недельных колонок.

**Формат заголовка:**
```
25.01
───────
ЗП
```
или
```
10.02
───────
ЗП
```
Под датой мелким шрифтом — «ЗП» (для всех колонок).

**Содержимое ячейки ЗП-колонки (для каждого проекта):**
- Показывается **распределённая доля** зарплатной выплаты, приходящаяся на этот проект.
- Расчёт: `getProjectSalaryOnDate(projectId, date)` (подробнее в разделе 4).
- Текст выводится **курсивом, серым цветом** — это справочная информация.
- Эта сумма **НЕ учитывается** в строке «Итого» (чтобы не было двойного списания).

**В строке «Итого»:** показывается общая сумма ЗП-выплаты на эту дату: `getSalaryPaidOnDate(date)`.

#### 3.1.7. Строки проектов

Каждая строка проекта содержит:
- **Название проекта** (кликабельное → переход на вкладку «Финансы» карточки проекта)
- Кнопка **«+»** — быстрое создание бухгалтерской задачи с авто-привязкой к проекту
- Значения аналитических столбцов
- Значения по неделям (расходы/приходы)
- Значения по ЗП-колонкам (справочно)

#### 3.1.8. Раздел «Вне проекта»

Внизу таблицы, после всех проектов, располагается раздел **«Вне проекта»**.

Здесь отображаются **только бухгалтерские задачи без привязки к проекту** — каждая задача отдельной строкой.

**Важно:** ЗП и авансы в этом разделе НЕ отображаются. ЗП обрабатывается через систему ведомостей (вкладка «Выплаты ЗП»).

**Особенность диалога выплаты для «Вне проекта»:**
- При клике на ячейку в строке задачи «Вне проекта» открывается диалог добавления выплаты.
- Поле **«Задача»** автоматически заполняется названием задачи и **заблокировано** (нельзя изменить).
- Поле **«Сумма»** предзаполняется суммой из бюджета/amount задачи, выводится **курсивом**. Как только пользователь начинает редактировать сумму — курсив снимается.

#### 3.1.9. Строка «Итого»

Последняя строка таблицы — **«Итого»** (жирный шрифт, фон `bg-muted`).

Содержит суммы по каждому столбцу:
- Аналитические столбцы: сумма по всем проектам.
- Недельные столбцы: сумма расходов по всем проектам + «Вне проекта»; сумма приходов по всем проектам.
- ЗП-колонки: общая сумма ЗП-выплаты (`getSalaryPaidOnDate`).

#### 3.1.10. Диалог добавления выплаты / прихода

Открывается при клике на ячейку (только для роли «Бухгалтер»).

**Поля диалога:**
| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| Вкладки «Расход / Приход» | Tabs | — | Переключатель типа операции |
| Задача | Select (выпадающий список) | Нет | Список бухгалтерских задач, привязанных к этому проекту. Для «Вне проекта» — заблокировано с авто-заполнением |
| Сумма | Number input | Да | Сумма в рублях. Для «Вне проекта» предзаполняется из задачи |
| Примечание | Text input | Нет | Свободный текст |

**Кнопки:** «Сохранить», «Отмена»

**Назначение:** Показать, из бюджетов каких проектов складывается зарплата каждого сотрудника на основании данных учёта рабочего времени.

#### 3.2.1. Фильтры

| Фильтр | Тип | По умолчанию |
|---|---|---|

#### 3.2.2. Основная таблица

Группировка по сотрудникам. Бухгалтеры **не отображаются** в списке.

**Строка сотрудника (жирная, фон `bg-muted/30`):**
| Столбец | Значение |
|---|---|
| Сотрудник | ФИО (кликабельное → открывает табель рабочего времени) |
| Подразделение | Название подразделения |
| Тип | Badge: «Штат» или «Контракт» |
| Проект | — (пусто) |
| Дней | Общее кол-во рабочих дней (для контрактников — «—») |
| Доля | 100% |
| Ставка | Дневная ставка или контрактная сумма |
| Начислено | Итого начислено за месяц |

**Подстроки проектов (под каждым сотрудником):**
| Столбец | Значение |
|---|---|


#### 3.3.3. Таблица сотрудников

| Столбец | Описание |
|---|---|
| ☑ | Чекбокс «Выплачено» (заголовок = «Выбрать всех») |
| Сотрудник | ФИО (кликабельное → открывает табель) |
| Подразделение | Название подразделения |
| Тип | Badge: «Штат» / «Контракт» |
| Начислено (мес.) | Полная сумма начислений за месяц (серым) |
| К выплате | Сумма к выплате (редактируемый Input, если ведомость не проведена) |

**Если ведомость проведена:** все поля только для чтения, чекбоксы заблокированы.

**Клик по имени сотрудника:** открывается диалог `TimeTrackingDialog`.

#### 3.3.4. Кнопка «Провести выплату»

Внизу таблицы. Доступна только если ведомость не проведена и хотя бы один сотрудник отмечен.

При клике — диалог подтверждения:
- Тип ведомости и месяц
- Сумма к выплате
- Кол-во отмеченных сотрудников
- Остаток невыплаченных (красным, если есть)

После подтверждения:
- `processed = true`
- Данные сохраняются в глобальное хранилище
- Суммы автоматически появляются в соответствующих ЗП-колонках основной таблицы

## 4. Расчёт зарплаты и аванса (подробная логика)

### 4.1. Фиксированные даты

| Дата | Тип | Описание |
|---|---|---|
| **25-е число** текущего месяца | Аванс | 50% от начислений за текущий месяц |
| **10-е число** следующего месяца | ЗП (остаток) | Начислено − Аванс |

Пример: за январь 2026 →
- Аванс: 25 января 2026 (50% от начислений за январь)
- ЗП: 10 февраля 2026 (остаток за январь)

### 4.2. Расчёт начислений сотрудника за месяц

#### Штатный сотрудник:
```
начислено = Σ (дней_на_проекте × дневная_ставка) по всем проектам
```

Источник данных «дней на проекте» — модуль «Учёт рабочего времени» (таблица time entries).

#### Контрактный сотрудник:
```
начислено = контрактная_ставка (фиксированная сумма)
```

### 4.3. Расчёт суммы аванса

```
аванс = ROUND(начислено × 0.5)
```

### 4.4. Расчёт суммы ЗП (остаток)

```
зп = начислено − аванс
```

### 4.5. Распределение выплаты по проектам

Когда ведомость проведена, каждая выплата сотрудника распределяется по проектам пропорционально отработанным дням.

#### Для штатных сотрудников:
```
доля_проекта = дней_на_проекте / общее_кол-во_дней_сотрудника
сумма_на_проект = ROUND(выплата_сотрудника × доля_проекта)
```

#### Для контрактных сотрудников:
```
кол-во_проектов = количество проектов, на которых числится сотрудник
сумма_на_проект = выплата_сотрудника / кол-во_проектов
```

### 4.6. Определение месяца для расчёта

Для ведомости с типом `advance` (25-е число):
```
месяц_расчёта = месяц даты ведомости (текущий)
```

Для ведомости с типом `salary` (10-е число):
```
месяц_расчёта = месяц даты ведомости − 1 (предыдущий)
```

### 4.7. Ведомости (Payroll)

Структура ведомости:
```typescript
interface SalaryPayroll {
  id: string;                        // Уникальный ID, формат: "payroll-YYYY-MM-DD"
  date: string;                      // Дата выплаты, формат: "YYYY-MM-DD"
  type: "salary" | "advance";        // "advance" = аванс (25-е), "salary" = ЗП (10-е)
  monthLabel: string;                // Метка для отображения, напр. "Январь 2026"
  employees: SalaryPaymentRecord[];  // Список сотрудников с суммами
  processed: boolean;                // Проведена ли ведомость
}

interface SalaryPaymentRecord {
  employeeId: string;
  employeeName: string;
  amount: number;      // Сумма к выплате (может быть отредактирована бухгалтером)
  paid: boolean;       // Отмечен ли как выплаченный
}
```

### 4.8. Отображение ЗП в основной таблице

**В ЗП-колонках (25-е и 10-е число):**
- Для каждого проекта: `getProjectSalaryOnDate(projectId, date)` — распределённая доля
- Выводится курсивом, серым текстом
- **НЕ включается** в строку «Итого» проектов (чтобы избежать двойного подсчёта)

**В строке «Итого»:**
- `getSalaryPaidOnDate(date)` — полная сумма выплаты по ведомости

### 4.9. Какие сотрудники включаются в расчёт

- Все сотрудники, **кроме бухгалтеров** (роль `accountant` исключается).
- Включаются только сотрудники с начислениями > 0 за соответствующий месяц.

## 5. Вкладка «Финансы» в карточке проекта

**Расположение:** Карточка проекта → вкладка «Финансы»  
**Доступ:** Администратор, ГИП, Бухгалтер  
**Редактирование:** только Бухгалтер

### 5.1. Сводные карточки (верхняя часть)

| Карточка | Формула |
|---|---|
| Бюджет проекта | `project.budget` |
| Всего выплачено | Сумма всех `payments` по проекту |
| Остаток | Бюджет − Выплачено |

### 5.2. Кнопка «Создать задачу»

Открывает диалог создания бухгалтерской задачи:
- Тип задачи автоматически = «Бухгалтерская» (заблокирован).
- Проект = текущий проект (заблокирован).
- Доступна для: Администратор, ГИП, Бухгалтер.

### 5.3. Подвкладки

#### 5.3.1. Финансы (журнал транзакций)

Таблица всех финансовых операций по проекту:

| Столбец | Описание |
|---|---|
| Неделя | Дата начала недели (dd.MM) |
| Категория | Тип: ЗП/Аванс, Субподрядчики, Доп. затраты, Другое |
| Связанная задача | Название бухгалтерской задачи |
| Сумма | Сумма транзакции |

Строка **«Итого выплачено»** внизу.

#### 5.3.2. Учёт времени (затраты на персонал)

| Столбец | Описание |

#### 5.3.3. Субподрядчики/фриланс

| Столбец | Описание |
|---|---|
| Субподрядчик | Название организации/ИП |

## 6. Бухгалтерские задачи

### 6.1. Подтипы

| Подтип | Описание |
|---|---|
| Субподрядчики / Фриланс | Оплата внешним подрядчикам |
| Доп. затраты | Командировки, материалы, лицензии и прочее |
| Другое | Прочие финансовые операции |

> **Важно:** Подтип «Зарплата / Аванс» **удалён** из интерфейса. Выплаты ЗП обрабатываются через систему ведомостей (вкладка «Выплаты ЗП»).

### 6.2. Привязка к проекту

- Задача **может** быть привязана к проекту → отображается в финансах этого проекта.
- Задача **может** быть без проекта → отображается в разделе «Вне проекта» основной таблицы.
- При создании из карточки проекта → проект подставляется автоматически.
- При создании из таблицы «Финансы» (кнопка «+» в строке проекта) → проект подставляется автоматически.

### 6.3. Создание бухгалтерской задачи

**Кто может создать:**
- **Бухгалтер** — может создавать как проектные, так и внепроектные задачи.
- **Администратор, ГИП** — могут создавать только проектные бухгалтерские задачи (с обязательной привязкой к проекту).

**Форма создания:**
| Поле | Тип | Описание |
|---|---|---|
| Название | Text | Название задачи |
| Тип задачи | Select (заблокирован) | «Бухгалтерская» |
| Подтип | Select | Субподрядчики, Доп. затраты, Другое |
| Проект | Select | Привязка к проекту (авто-заполняется если создаётся из проекта) |
| Исполнитель | Select | Бухгалтер-исполнитель |
| Сумма | Number | Бюджет задачи |
| Сроки | Date range | Плановые даты |

**Для подтипа «Субподрядчики»:**
- Доступен выбор конкретных исполнителей (субподрядчиков) из списка.
- Отображаются ставки и рабочие дни.
- Кнопка «Выбрать всех».

### 6.4. Отображение в таблице «Финансы»

- Проектные задачи → выплаты появляются в строке соответствующего проекта.
- Внепроектные задачи → выплаты появляются в разделе «Вне проекта» (каждая задача — отдельная строка).

## 7. Уведомления

### 7.1. Индикатор уведомлений (колокольчик)

**Доступ:** Бухгалтер, Администратор, ГИП.

**Источники уведомлений:**
1. **Активные бухгалтерские задачи** — задачи со статусом `new` или `in_progress`, где текущий пользователь является исполнителем.
2. **Просроченные планируемые приходы** — если в режиме «План» на прошедшую дату запланирован приход, но фактического прихода нет.

**Бейдж на колокольчике:** число = кол-во активных задач + кол-во просроченных приходов.

**Содержимое выпадающего меню:**

Секция «Задачи»:
- Список задач (название, дата)
- Клик → переход к задаче

Секция «Нет прихода»:
- ⚠️ Иконка предупреждения
- Название проекта
- Неделя (dd.MM)
- Ожидаемая сумма

### 7.2. Логика определения «Нет прихода»

Для каждого проекта и каждой прошедшей недельной колонки:
1. Проверить: есть ли запланированный приход (`plannedIncomes[projectId][columnIndex]`)?
2. Если да — проверить: есть ли фактический приход (`incomes[projectId][weekIndex]`)?
3. Если фактический приход = 0 или отсутствует → создать уведомление.

**В режиме «Факт»:** в ячейке показывается текст «Нет прихода» + иконка «глаз» для скрытия.
**В режиме «План»:** ячейка подсвечивается красным фоном.

---

## 8. Режим «Факт / План»

### 8.1. Переключатель

Расположен над таблицей. Два режима:
- **Факт** (по умолчанию) — фактические данные.
- **План** — планируемые приходы.

### 8.2. Режим «Факт»

- Ячейки недель показывают фактические расходы и приходы.
- Бухгалтер может добавлять выплаты и приходы.

### 8.3. Режим «План»

- В ячейках недель показываются **только планируемые приходы** (нижняя строка).
- Расходы не отображаются.
- Бухгалтер может добавлять/редактировать/удалять планируемые приходы.
- Каждый планируемый приход: `{ id, title, amount }`.

### 8.4. Добавление планируемого прихода

При клике на ячейку в режиме «План»:
- Диалог с полями: Название, Сумма.
- Сохранение в `plannedIncomes[projectId][columnIndex]`.

### 8.5. Индикация просроченных приходов

Если дата недели уже прошла и планируемый приход есть, но фактического прихода нет:
- **В режиме «План»:** ячейка с красным фоном.
- **В режиме «Факт»:** текст «Нет прихода» с иконкой «глаз» (при клике скрывается для этой ячейки).
