## Общее описание

Модуль «Финансы» предназначен для учёта и контроля финансовых потоков по проектам организации. Он объединяет:

- **Поступления** — входящие платежи от заказчиков.
- **Операционные расходы** — оплата субподрядчикам, дополнительные затраты.
- **ФОТ** — фонд оплаты труда, рассчитанный на основе учёта рабочего времени.
- **Сальдо** — автоматический расчёт остатков и баланса.
- **Выплаты ЗП** — ведомости на фиксированные даты (5-е и 20-е числа).
- **Распределение ЗП** — анализ начислений по проектам на основе табеля.

## Структура модуля

Модуль состоит из трёх вкладок:

### 1. Финансы (основная таблица)
Глобальная сводная таблица по всем проектам за текущее полугодие. Содержит:
- Переключатель **Факт / План** — «Факт» показывает реальные данные, «План» — в разработке.
- Управление **видимостью столбцов** — кнопка «Столбцы X/7» позволяет скрывать/показывать сводные столбцы.

### 2. Выплаты ЗП
Рабочее место бухгалтера для проведения зарплатных ведомостей на фиксированные даты (5-е и 20-е числа каждого месяца).

### 3. Распределение ЗП
Анализ начислений на основе учёта рабочего времени — показывает, из бюджетов каких проектов складывается зарплата каждого сотрудника.

## Сводная таблица

**Расположение:** Боковое меню → «Финансы» → вкладка «Финансы»  
**Доступ:** Администратор, Бухгалтер

### Сводные столбцы (7 шт., настраиваемые)

| № | Столбец | Описание | Формула |
|---|---------|----------|---------|
| 1 | **Сумма по договору** | Общий бюджет проекта | — |
| 2 | **Поступления** | Суммы входящих платежей | Σ приходов |
| 3 | **Сальдо (Остаток к поступлению)** | Сколько ещё должен заказчик | Бюджет − Поступления |
| 4 | **Операционные расходы** | Расходы без учёта ЗП | Σ выплат (субподряд, доп. затраты и др.) |
| 5 | **ФОТ** | Фонд оплаты труда | Рассчитывается из табеля (дни × ставка) |
| 6 | **Общие расходы** | Все расходы по проекту | Операционные + ФОТ |
| 7 | **Сальдо (Между поступлениями и расходами)** | Финансовый результат | Поступления − Общие расходы |

Каждый столбец можно скрыть/показать через кнопку **«Столбцы»**.

### Временные столбцы

После сводных столбцов идут еженедельные столбцы за текущее полугодие, сгруппированные по месяцам.

В каждой ячейке даты отображаются:
- **Зелёным** (+) — суммы поступлений
- **Красным** (−) — суммы расходов
- Оба значения могут быть в одной ячейке одновременно
- **⚠ «нет прихода»** — жёлтое предупреждение, если в ячейке есть расход, но нет прихода

### Цветовая индикация

| Элемент | Цвет | Значение |
|---------|------|----------|
| Сальдо (Остаток к поступлению) | 🟢 Зелёный | Всё получено (≤ 0) |
| Сальдо (Остаток к поступлению) | 🔴 Красный | Ещё не получено (> 0) |
| Сальдо (Баланс) | 🟢 Зелёный | Положительный баланс |
| Сальдо (Баланс) | 🔴 Красный | Отрицательный баланс |
| Ячейка даты | 🟡 Жёлтый ⚠ | Есть расход без прихода |

### Раздел «Вне проекта»

Внепроектные бухгалтерские задачи (ЗП, хоз. расходы и др.) отображаются отдельными строками. Сводные столбцы для них показывают «—».

### Строка «Итого»

Суммирует все проекты. ФОТ суммируется по всем проектам, операционные расходы включают внепроектные.

## Вкладка «Выплаты ЗП»

**Расположение:** Страница «Финансы» → вкладка «Выплаты ЗП»  
**Доступ:** Администратор, Бухгалтер

### Назначение

Рабочее место бухгалтера для проведения зарплатных ведомостей. Выплаты ЗП привязаны к фиксированным датам:
- **20-е число** — Аванс (50% от начисления за текущий месяц)
- **5-е число** — ЗП (остаток за предыдущий месяц)

### Порядок работы

1. Выберите ведомость из выпадающего списка (по умолчанию — первая непроведённая).
2. Проверьте суммы к выплате (рассчитаны автоматически из табеля).
3. При необходимости скорректируйте сумму вручную.
4. Отметьте галочками сотрудников, которым произведена выплата.
5. Нажмите **«Провести выплату»** и подтвердите.

### Сводные карточки

| Карточка | Описание |
|----------|----------|
| Итого к выплате | Общая сумма ведомости |
| Выплачено | Сумма по отмеченным сотрудникам |
| Остаток | Сколько ещё не выплачено |

### Статусы ведомости

- **Не проведена** — можно редактировать суммы и отмечать выплаты.
- **Проведена** — данные зафиксированы, изменения невозможны.

## Вкладка «Распределение ЗП»

**Расположение:** Страница «Финансы» → вкладка «Распределение ЗП»  
**Доступ:** Администратор, Бухгалтер

### Назначение

Показывает начисления на основе учёта рабочего времени: из бюджетов каких проектов складывается зарплата каждого сотрудника.

### Фильтры

- **Месяц** — выбор периода (по умолчанию текущий месяц).
- **Подразделение** — фильтрация по отделам.
- **Поиск** — поиск по имени сотрудника.

### Структура таблицы

| Столбец | Описание |
|---------|----------|
| Сотрудник | ФИО (жирная строка-сводка) |
| Подразделение | Отдел сотрудника |
| Тип | «Штат» или «Контракт» |
| Проект | Название проекта (подстроки) |
| Дней | Отработанные дни на проекте |
| Доля | Процент от общего времени |
| Ставка | Дневная ставка или сумма контракта |
| Начислено | Сумма к начислению |

### Сводка по проектам

Внизу — таблица нагрузки ЗП на бюджет каждого проекта.

## Бухгалтерские задачи

### Подтипы задач

| Подтип | Описание |
|--------|----------|
| Зарплата / Аванс | Выплата ЗП или аванса (только вне проекта) |
| Субподрядчики / Фриланс | Оплата внешним подрядчикам |
| Доп. затраты | Командировки, материалы, лицензии |
| Другое | Прочие финансовые операции |

### Привязка к проекту

- Задачи «Зарплата / Аванс» — **только вне проекта**.
- Остальные подтипы могут быть привязаны к проекту.

## Инструкции по ролям

### 👤 Бухгалтер

#### Ежемесячный цикл

1. **Контроль поступлений** — фиксируйте приходы от заказчиков в ячейках таблицы «Финансы».
2. **Операционные расходы** — вносите выплаты субподрядчикам и доп. затраты через диалог в ячейках.
3. **Выплата аванса (20-е число):**
   - Перейдите на вкладку «Выплаты ЗП».
   - Выберите ведомость аванса.
   - Проверьте суммы (50% от начислений).
   - Отметьте сотрудников и проведите ведомость.
4. **Выплата ЗП (5-е число):**
   - Аналогично, но выберите ведомость ЗП (остаток за прошлый месяц).
5. **Контроль сальдо** — следите за столбцами «Сальдо» в таблице «Финансы».

#### Управление столбцами

- Используйте кнопку **«Столбцы»** для скрытия ненужных сводных столбцов.
- Переключатель **Факт / План** позволяет переключаться между режимами (План в разработке).

### 👤 Администратор

- Просматривает все проекты и финансовые данные.
- Может создавать бухгалтерские задачи.
- Использует столбцы «Сальдо» для контроля финансового состояния проектов.

### 👤 ГИП

- Видит финансы только своих проектов через вкладку «Финансы» в карточке проекта.
- Может создавать бухгалтерские задачи для своих проектов.
